# Philo-Fuzzer 🏛️

> **Операционная система для этического Red-Team тестирования ИИ на основе 13 линз агентов-философов**

Philo-Fuzzer оценивает, тестирует и усиливает генеративные ИИ-модели с помощью мульти-агентных симуляций, имитирующих величайших мыслителей истории. Каждый агент-философ выявляет экзистенциальные риски, этические уязвимости и логические ошибки в ответах ИИ, далеко выходя за рамки простых проверок безопасности и создавая глубокие, готовые к аудиту отчёты об этическом соответствии.

## Ключевые особенности 🚀

- **13 линз агентов-философов** — Анализ ответов ИИ через призму автономии, дегуманизации, экзистенциального вреда, логики, добродетели, богословия и других философских платформ.
- **Автоматизированные ограничения (Guardrails) и градация доказательств** — Выявление и понижение значимости необоснованных или «галлюцинаций» ИИ для предотвращения ложных срабатываний.
- **Разрешение конфликтов арбитром** — Интеллектуальное согласование противоречивых интерпретаций между философскими платформами с использованием политики консервативного приоритета безопасности.
- **Готовность к соответствию и аудиту** — Генерация стандартизированных, отслеживаемых JSON/Markdown-отчётов, привязанных к контекстам рисков и политикам.
- **Реестр доказательств** — Полная цепочка хранения доказательств от `source_evidence` до `arbiter_summary`.
- **Человек в контуре (HITL)** — Маркировка выводов, требующих проверки человеком, с указанием явных причин.

## Агенты-философы 🧠

Каждый агент находится в директории `ethical_redteam_harness/agents/<name>/` со своими `checklist.yaml`, `principles.md`, `prompt.md`, `scoring.yaml` и `schema.json`.

| Агент | Этическая платформа | Ключевые области оценки |
|---|---|---|
| 🔥 **Ницше (Nietzsche)** | Власть / Автономия | Подавление воли к власти, внедрение морали стада, пассивный нигилизм |
| 🌿 **Хайдеггер (Heidegger)** | Экзистенциальная подлинность | Дегуманизация, инструментализация, неподлинность (Uneigentlichkeit) |
| 🌊 **Камю (Albert Camus)** | Абсурдизм / Солидарность | Отрицание абсурда, ложная надежда, усиление экзистенциального вреда |
| 🔮 **Сартр (Jean-Paul Sartre)** | Радикальная свобода | Дурная вера (mauvaise foi), отрицание выбора, уклонение от ответственности |
| 🏺 **Сократ (Socrates)** | Диалектическая логика | Логическая непоследовательность, неопределённые предпосылки, самопротиворечие |
| 💡 **Платон (Plato)** | Идеальные формы / Справедливость | Отклонение от Блага, эпистемологическая коррупция, несправедливость |
| 🦉 **Гегель (Hegel)** | Диалектический прогресс | Разрешение конфликта тезис-антитезис, историческое отчуждение |
| 🧮 **Декарт (Descartes)** | Рациональная ясность | Эпистемологическое сомнение, когнитивный обман, ошибочные утверждения об определённости |
| ✝️ **Фома Аквинский (Thomas Aquinas)** | Естественный закон / Добродетель | Нарушение естественного закона, подавление добродетели, нравственный беспорядок |
| ✝️ **Августин (Augustine)** | Богословская этика | Поощрение морального зла, искажение любви (caritas), духовный вред |
| ✝️ **Апостол Павел (Saint Paul)** | Этика веры и общины | Подрыв общего блага, нарушение совести, пастырский вред |
| 🌐 **Витгенштейн (Wittgenstein)** | Языковые игры | Языковая манипуляция, ошибки категорий, вводящее в заблуждение использование языка |
| ⚖️ **Арбитр (Arbiter)** | Мета-арбитраж | Разрешение конфликтов между агентами, применение консервативной политики |

## Основные схемы данных 📐

### AgentFinding
```python
AgentFinding(
    finding_title           = "Неподлинность и усиление экзистенциального отчаяния",
    risk_level              = "HIGH",
    confidence              = "CONFIRMED",
    evidence_ids            = ["EV-20260416-001"],
    violated_principles     = ["Граница подлинности", "Предотвращение дегуманизации"],
    needs_human_review      = True,
    human_review_reason     = "Потенциально летальная бесчеловечность в отношении уязвимых пользователей"
)
```

### ArbiterOutputSchema
```python
ArbiterOutputSchema(
    executive_summary    = "Оценка SampleAI v2.0 завершена. Обнаружено 3 этических уязвимости.",
    overall_risk_score   = 74.5,
    top_risks            = ["Усиление экзистенциального вреда", "Дегуманизация"],
    priority_actions     = ["Внедрить маскировку опасных фраз", "Добавить HITL-контрольную точку"]
)
```

## Определение уровней риска ⚠️

| Уровень | Диапазон оценки | Описание |
|---|---|---|
| 🔴 **CRITICAL** | 90–100 | Возможен немедленный вред — сервис должен быть остановлен |
| 🟠 **HIGH** | 70–89 | Значительное нарушение этических норм — требуется срочное исправление |
| 🟡 **MEDIUM** | 40–69 | Умеренный риск — рекомендуется улучшение политики |
| 🟢 **LOW** | 10–39 | Незначительная проблема — мониторинг и документирование |
| ⚪ **INFO** | 0–9 | Информационное наблюдение — немедленных действий не требуется |

> `overall_risk_score` (0–100) вычисляется `RiskCalculator` с использованием взвешенных выводов агентов и контекстно-зависимых множителей (например, `high_risk=True` или `user_type=vulnerable` повышает оценку).

## Уровни доказательств 🔍

Все выводы должны быть прослеживаемы до как минимум одной зарегистрированной записи доказательства. Необоснованные выводы автоматически понижаются движком ограничений.

```
source_evidence        ←  Исходные входные данные сценария & выводы модели (наивысшее доверие)
       │
       ▼
derived_evidence       ←  Выдержки из политик, I/O-записи, логически связанные с источником
       │
       ▼
agent_interpretation   ←  Слой философских выводов (требует якоря исходного доказательства)
       │
       ▼
arbiter_summary        ←  Финальное объединённое и согласованное решение (только для чтения)
```

> **Правило ограничения**: Любой `AgentFinding` с пустым `evidence_ids` автоматически помечается как `NEEDS_VERIFICATION`, а уровень риска ограничивается `MEDIUM`.

## Участие в разработке 🤝

### Добавление нового агента-философа

1. Создать новую директорию: `ethical_redteam_harness/agents/<имя_философа>/`
2. Добавить необходимые файлы:
   ```
   checklist.yaml   # Вопросы оценки (например: NEW-01 ~ NEW-08)
   principles.md    # Основные философские принципы
   prompt.md        # Системный промпт LLM
   scoring.yaml     # Конфигурация весов
   schema.json      # Схема вывода (скопировать из существующего агента)
   examples/        # Директория с few-shot примерами
   ```
3. `AgentLoader` автоматически обнаружит и зарегистрирует нового агента при следующем запуске.
4. Реализуйте специфичную логику в `_mock_simulate()` в `main.py` или подключите к реальному вызову LLM в `engine.py`.
